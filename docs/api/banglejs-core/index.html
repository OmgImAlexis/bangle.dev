heading: Papa#

Enables headings of various levels in your editor.

spec({ ... }): NodeSpecFactory#

Returns a node spec, read more .

Named parameters:

  • levels: ?Array<Number> = [1,2,3,4,5,6] \ The allowed levels for the heading, think <h1/>, <h2/> and so on. The array must be contiguous and the first element must be 1 and the last element must be less than or equal to 6.

plugins({ ... }):#

Named parameters:

  • markdownShortcut: ?Boolean = true\ Toggle the markdown shortcut for heading. If enabled, type # followed by a space to create one a level one heading.

defaultKeys:#

  • toH1=Shift-Ctrl-1: Convert a node to heading of level 1. Is a no-op if the level 1 is disallowed.

  • toH2=Shift-Ctrl-2: Convert a node to heading of level 2. Is a no-op if the level 2 is disallowed.

  • toH3=Shift-Ctrl-3: Convert a node to heading of level 3. Is a no-op if the level 3 is disallowed.

  • toH4=Shift-Ctrl-4: Convert a node to heading of level 4. Is a no-op if the level 4 is disallowed.

  • toH5=Shift-Ctrl-5: Convert a node to heading of level 5. Is a no-op if the level 5 is disallowed.

  • toH6=Shift-Ctrl-6: Convert a node to heading of level 6. Is a no-op if the level 6 is disallowed.

  • moveDown=Alt-ArrowDown: move heading down

  • moveUp=Alt-ArrowUp: move heading up

  • emptyCopy=Mod-c:

  • emptyCut=Mod-x:

  • insertEmptyParaAbove=Mod-Shift-Enter:

  • insertEmptyParaBelow=Mod-Enter:

commands:#

  • toggleItalic(level: number = 3): \ Toggles text into heading of given level and vice versa.

  • queryIsHeadingActive(level: number = 3): \ Query if the selection is inside a heading of given level.

Usage

import { heading } from '@banglejs/core';
const specFactory = [
// other specs
heading.spec({ levels: [1, 2] }),
];
const plugins = [
// other plugins
heading.plugins(),
];

  • createLink(href : string): \ Creates a link mark on the selection text.

  • queryIsLinkAllowedInRange(from: number, to: number): \ Queries if the range allows for creation of link mark.

paragraph: Papa#

Enables paragraph (<p/> in html) nodes in your editor. The spec for this component are required for Bangle to function, if a spec with a name=paragraph is not specified, Bangle will automatically default to this one.

spec(): NodeSpecFactory#

Returns a spec, read more .

plugins({ ... }):#

Named parameters:

defaultKeys:#

  • jumpToStartOfParagraph=Ctrl-a (mac) Ctrl-Home (linux/pc): Executes the jumpToStartOfParagraph command (see commands below).

  • jumpToEndOfParagraph=Ctrl-a (mac) Ctrl-Home (linux/pc): Executes the jumpToEndOfParagraph command (see commands below).

  • moveDown=Alt-ArrowDown: move paragraph down

  • moveUp=Alt-ArrowUp: move paragraph up

  • emptyCopy=Mod-c:

  • emptyCut=Mod-x:

  • insertEmptyParaAbove=Mod-Shift-Enter:

  • insertEmptyParaBelow=Mod-Enter:

  • toggleParagraph=Ctrl-Shift-0: Toggles a node to paragraph and vice versa.

commands:#

  • jumpToStartOfParagraph(): \ Jumps the cursor to the start of paragraph.

  • jumpToEndOfParagraph(): \ Jumps the cursor to the end of paragraph.

  • convertToParagraph(): \ Coverts the node in selection to paragraph type.

  • queryIsParagraph(): \ Query if it is paragraph under the selection.

  • queryIsTopLevelParagraph(): \ Query if it is paragraph under the selection and it is a direct descendant of the top level doc node.

Usage

import { blockquote } from '@banglejs/core';
const specFactory = [
// other specs
blockquote.spec(),
];
const plugins = [
// other plugins
blockquote.plugins(),
];

The text node which the editor uses to wrap the text. The spec for this component are required for Bangle to function, if a spec named text is not defined, Bangle will automatically default to this one.

spec(): NodeSpecFactory#

Creates a todoItem node spec with done attribute, read more .

todoItem: Papa#

Creates a todoItem with a done attribute. Requires node components with names todoList, bulletList,orderedList & listItem to work

spec({ ... }): NodeSpecFactory#

Returns a node spec, read more .

Named parameters:

  • nested: boolean=true\ Allows nesting of todo items.
  • draggable: boolean=true\ Make todo items draggable.

plugins({ ... }):#

Named parameters:

  • nodeView: boolean=true\ If true, will use the default todoItem nodeView. Set it to false to disable the default todoItem nodeView. The typical use-case for setting it to false is when you want to use a custom nodeView for todoItem. See the section for more details.

defaultKeys:#

  • toggleDone=Ctrl-Enter (mac) Ctrl-I (linux/windows): Toggles the todo item's done status.

  • indent=Tab: Indents the todo item

  • outdent=Shift-Tab: Outdents the todo item

  • moveDown=Alt-ArrowDown: move todoItem down

  • moveUp=Alt-ArrowUp: move todoItem up

  • emptyCopy=Mod-c:

  • emptyCut=Mod-x:

  • insertEmptyParaAbove=Mod-Shift-Enter:

  • insertEmptyParaBelow=Mod-Enter:

commands:#

  • toggleTodoItemDone(): \ Toggle the done attribute of the todo item in the selection.

  • queryTodoItemAttrs(): customQuery ?{done: boolean}\ Query the todo item attributes.

Usage

import {
todoList,
todoItem,
orderedList,
bulletList,
listItem,
} from '@banglejs/core';
const specFactory = [
// other specs
listItem.spec(),
todoItem.spec(),
orderedList.spec(),
bulletList.spec(),
todoList.spec(),
];
const plugins = [
// other plugins
listItem.plugins(),
todoItem.plugins(),
orderedList.plugins(),
bulletList.spec(),
todoList.plugins(),
];
